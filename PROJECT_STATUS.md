## 🔧 已修复的关键 Bug

| Bug | 原因 | 修复方案 |
|-----|------|---------|
| 真机请求超时（433） | MiniMax 模型两步走耗时 49 秒，SCF 30 秒超时 | 切换为 Kimi Coding Plan Anthropic 格式，一步走 5 秒 |
| 框选挖空显示乱码 | `clearRect+drawImage` 坐标映射错误 | 改为四周矩形遮罩 |
| 云函数代码未更新 | zip 文件未重新打包，上传了旧版本 | 重新打包 zip 后部署 |
| API Key 无效 | 环境变量更新失败，函数状态冲突 | 等待函数就绪后单独更新配置 |
| 图片未传到 Kimi | `req.write(data); req.end()` 大数据截断 | 改为 `req.end(data)` 完整发送 |
| k2.6 思维链导致超时/截断 | k2.6 输出大量 reasoning_content | 切换为 `moonshot-v1-32k-vision-preview` |
| 横屏强制 landscape 布局错乱 | `pageOrientation: landscape` 导致 windowHeight 极小 | 去掉 landscape，改为自动旋转竖版照片 |
| JSON 嵌套解析失败 | prompt 中的 JSON 示例被模型当作输出内容 | 去掉 JSON 示例，改用 `response_format: json_object` |
| 结果页按钮不对齐 | `.btn-primary` 样式未定义 | 统一按钮高度和 flex 布局 |
| "再拍一题"按钮失效 | `wx.switchTab` 只能跳 tabBar 页面 | 改为 `wx.reLaunch` |
| 历史记录显示为空 | 新格式 `question` 在 `questions[0]` 中 | `saveToHistory` 时自动提取到顶层 |
| GitHub Push Protection 阻止 | 代码中硬编码腾讯云密钥 | 全部改为从 `.env` 环境变量读取 |

## 🏗️ 技术架构

```
微信小程序 (前端)
├── 拍照/选图 → 压缩 → Canvas 框选 → 裁剪 → Base64
└── 结果展示：swiper 多题 + 原题解析 + 答案/解析折叠

腾讯云 SCF (后端)
├── Node.js 18，接收 Base64 图片
├── 调用 api.moonshot.cn/v1/chat/completions
└── 返回 JSON：original_question / questions[]

Kimi AI (模型)
├── moonshot-v1-32k-vision-preview
├── 支持图片输入（image_url base64）
└── response_format: json_object 强制输出 JSON
```

## ⚠️ 已知限制

1. **LaTeX 公式渲染**：小程序端未引入 KaTeX，数学公式以纯文本展示源码
2. **单张图片限制**：Kimi API 单张图片有大小限制，前端已压缩到 800px 最大边
3. **网络依赖**：必须使用函数 URL（HTTPS），真机需配置服务器域名白名单
4. **冷启动延迟**：SCF 首次调用可能有 1-3 秒冷启动，后续正常
5. **SVG 兼容性**：部分复杂 SVG 在小程序中渲染可能异常，需包含 `viewBox`
6. **题目错误检测依赖模型**：`original_error_analysis` 的准确性取决于模型判断，并非 100% 可靠
